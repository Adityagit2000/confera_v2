# Confera — Low Level Design (LLD)

This document contains the low-level code implementation details, schema structures, prompt definitions, and algorithms of the Confera SaaS platform.

---

## Section 1 — Database Schema & Data Models

The data layer is built on Supabase PostgreSQL with extensions `uuid-ossp` and `vector`. Row-Level Security (RLS) is enabled on all client-facing tables.

### 1. Table Definitions

#### Table: `profiles`
Tracks user details, onboarding state, and monthly mock interview allocations.
* **Columns**:
  * `id` `uuid` REFERENCES `auth.users(id)` PRIMARY KEY
  * `name` `text`
  * `onboarding_completed` `boolean` DEFAULT `false`
  * `interviews_used_this_month` `integer` DEFAULT `0`
  * `resume_analyses_used_this_month` `integer` DEFAULT `0`
  * `referral_code` `text` UNIQUE (Generated via trigger)
  * `is_founder` `boolean` DEFAULT `false`
  * `updated_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select/Update: `auth.uid() = id`

#### Table: `resumes`
Stores candidate resume files and raw parsed JSON details.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `user_id` `uuid` REFERENCES `profiles(id)` ON DELETE CASCADE
  * `ats_score` `integer`
  * `parsed_data` `jsonb`
  * `keywords_missing` `jsonb`
  * `original_filename` `text`
  * `file_url` `text`
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select/Insert/Delete: `auth.uid() = user_id`

#### Table: `interview_sessions`
Tracks mocks sessions.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `user_id` `uuid` REFERENCES `profiles(id)` ON DELETE CASCADE
  * `type` `text` (e.g. `dsa`, `system_design`, `hr`, `mckinsey_de`)
  * `status` `text` DEFAULT `'active'` (e.g. `active`, `completed`)
  * `transcript` `jsonb` DEFAULT `'[]'`
  * `job_role` `text`
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select/Insert/Update: `auth.uid() = user_id`

#### Table: `interview_answers`
Stores candidate's answers and coaching metrics evaluated per turn.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `session_id` `uuid` REFERENCES `interview_sessions(id)` ON DELETE CASCADE
  * `question` `text`
  * `answer_text` `text`
  * `score` `integer`
  * `tags` `jsonb` (contains filler words, logic scores, and coaching tips)
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select: `auth.uid() = (SELECT user_id FROM interview_sessions WHERE id = session_id)`

#### Table: `feedback_reports`
Final performance feedback reports generated at session completion.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `session_id` `uuid` REFERENCES `interview_sessions(id)` ON DELETE CASCADE UNIQUE
  * `overall_score` `integer`
  * `technical_score` `integer`
  * `communication_score` `integer`
  * `behavior_score` `integer`
  * `summary` `text`
  * `recommendations` `jsonb` (contains strengths, weaknesses, next steps)
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select: `auth.uid() = (SELECT user_id FROM interview_sessions WHERE id = session_id)`

#### Table: `transcript_embeddings`
Holds embedded Q&A blocks for the RAG search database.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `user_id` `uuid` REFERENCES `profiles(id)` ON DELETE CASCADE
  * `session_id` `uuid` REFERENCES `interview_sessions(id)` ON DELETE CASCADE
  * `content` `text` (Q&A plain text block)
  * `embedding` `vector(768)` (Generated via text-embedding-004)
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select/Insert: `auth.uid() = user_id`

#### Table: `prep_plans`
Stores the weekly study focus generated by the system.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `user_id` `uuid` REFERENCES `profiles(id)` ON DELETE CASCADE
  * `weekly_focus` `text`
  * `coaching_note` `text`
  * `daily_tasks` `jsonb` (contains Day 1-7 tasks)
  * `priority_interview_type` `text`
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select: `auth.uid() = user_id`

#### Table: `referrals`
Tracks candidate signups associated with referrers.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `referrer_id` `uuid` REFERENCES `profiles(id)` ON DELETE CASCADE
  * `referred_id` `uuid` REFERENCES `profiles(id)` ON DELETE CASCADE UNIQUE
  * `status` `text` DEFAULT `'pending'` (e.g. `pending`, `converted`)
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select: `auth.uid() = referrer_id OR auth.uid() = referred_id`

#### Table: `referral_earnings`
Stores conversion commissions.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `referrer_id` `uuid` REFERENCES `profiles(id)` ON DELETE CASCADE
  * `referred_id` `uuid` REFERENCES `profiles(id)` ON DELETE CASCADE
  * `amount` `numeric(10,2)`
  * `status` `text` DEFAULT `'pending'` (e.g. `pending`, `paid`)
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select: `auth.uid() = referrer_id`

#### Table: `subscriptions`
Stores customer active Pro plans.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `user_id` `uuid` REFERENCES `profiles(id)` ON DELETE CASCADE UNIQUE
  * `plan` `text` (e.g. `pro`)
  * `billing_cycle` `text` (e.g. `monthly`, `yearly`)
  * `status` `text` DEFAULT `'active'`
  * `start_date` `timestamp with time zone` DEFAULT `now()`
  * `end_date` `timestamp with time zone`
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * Select: `auth.uid() = user_id`

#### Table: `event_logs`
Tracks diagnostic logs and API events.
* **Columns**:
  * `id` `uuid` PRIMARY KEY DEFAULT `uuid_generate_v4()`
  * `user_id` `uuid` REFERENCES `profiles(id)` ON DELETE SET NULL
  * `event_type` `text`
  * `metadata` `jsonb`
  * `created_at` `timestamp with time zone` DEFAULT `now()`
* **RLS Policies**:
  * No public read/write allowed. Only readable by system service roles.

### 2. Custom PL/pgSQL Database Functions

#### `match_transcript_embeddings`
Calculates vector cosine similarity distances:
```sql
CREATE OR REPLACE FUNCTION match_transcript_embeddings(
  query_embedding vector(768),
  match_threshold float,
  match_count int,
  filter_user_id uuid
)
RETURNS TABLE (
  id uuid,
  content text,
  similarity float
)
LANGUAGE plpgsql ACCESS TO PUBLIC
AS $$
BEGIN
  RETURN QUERY
  SELECT
    te.id,
    te.content,
    1 - (te.embedding <=> query_embedding) AS similarity
  FROM transcript_embeddings te
  WHERE te.user_id = filter_user_id
    AND 1 - (te.embedding <=> query_embedding) > match_threshold
  ORDER BY te.embedding <=> query_embedding
  LIMIT match_count;
END;
$$;
```

#### `gift_pro_access`
Enables administrative gifting of Pro status to specified candidates:
```sql
CREATE OR REPLACE FUNCTION gift_pro_access(
  target_user_id uuid,
  duration_days int
)
RETURNS void
LANGUAGE plpgsql SECURITY DEFINER
AS $$
BEGIN
  INSERT INTO subscriptions (user_id, plan, billing_cycle, status, start_date, end_date)
  VALUES (
    target_user_id,
    'pro',
    'gifted',
    'active',
    now(),
    now() + (duration_days || ' days')::interval
  )
  ON CONFLICT (user_id) DO UPDATE
  SET status = 'active',
      billing_cycle = 'gifted',
      end_date = CASE 
                   WHEN subscriptions.end_date > now() THEN subscriptions.end_date + (duration_days || ' days')::interval
                   ELSE now() + (duration_days || ' days')::interval
                 END;
END;
$$;
```

#### `_shared/request-context.ts`
Authenticates incoming Bearer tokens, applies local rate limits, and matches prompt sanitizers:
```typescript
import { createClient } from '@supabase/supabase-js';

export async function authenticateRequest(req: Request) {
  const authHeader = req.headers.get('Authorization');
  if (!authHeader?.startsWith('Bearer ')) throw new Error('Unauthenticated');
  
  const token = authHeader.split(' ')[1];
  const supabaseClient = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_ANON_KEY')!,
    { global: { headers: { Authorization: authHeader } } }
  );

  const { data: { user }, error } = await supabaseClient.auth.getUser(token);
  if (error || !user) throw new Error('Invalid authentication token');
  return { user, supabaseClient };
}

export function sanitizeText(text: string): string {
  const blockedPatterns = [/system\s+override/i, /ignore\s+previous/i, /developer\s+mode/i];
  for (const pattern of blockedPatterns) {
    if (pattern.test(text)) throw new Error('Potential prompt injection detected.');
  }
  return text.replace(/[<>]/g, ''); // Sanitizes simple HTML
}
```

#### `_shared/ai-service.ts`
Orchestrates API fallback routing and tracks circuit breakers:
```typescript
class CircuitBreaker {
  private failures = 0;
  private state: 'closed' | 'open' | 'half-open' = 'closed';
  private cooldownUntil = 0;

  recordFailure() {
    this.failures++;
    if (this.failures >= 3) {
      this.state = 'open';
      this.cooldownUntil = Date.now() + 120000; // 2 minutes cooldown
    }
  }

  recordSuccess() {
    this.failures = 0;
    this.state = 'closed';
  }

  canExecute(): boolean {
    if (this.state === 'open') {
      if (Date.now() > this.cooldownUntil) {
        this.state = 'half-open';
        return true;
      }
      return false;
    }
    return true;
  }
}

const breakers: Record<string, CircuitBreaker> = {
  gemini: new CircuitBreaker(),
  groq: new CircuitBreaker(),
  openai: new CircuitBreaker(),
};

export async function executeAiCall(prompt: string, systemInstruction: string) {
  const providers = ['gemini', 'groq', 'openai'];
  for (const provider of providers) {
    const breaker = breakers[provider];
    if (!breaker.canExecute()) continue;

    try {
      let response = '';
      if (provider === 'gemini') {
        response = await callGeminiAPI(prompt, systemInstruction);
      } else if (provider === 'groq') {
        response = await callGroqAPI(prompt, systemInstruction);
      } else {
        response = await callOpenAIAPI(prompt, systemInstruction);
      }
      breaker.recordSuccess();
      return response;
    } catch (err) {
      console.warn(`[AI Service] ${provider} failed:`, err);
      breaker.recordFailure();
    }
  }
  throw new Error('All AI services are currently offline.');
}
```

### Core Endpoint Implementations

1. **`create-order`**: Calls the Razorpay API to initialize an order ID for billing cycles. Returns: `{ orderId, amount, currency }`.
2. **`verify-payment`**: Compares `HMAC-SHA256(razorpay_order_id + "|" + razorpay_payment_id, key_secret)` against `razorpay_signature`. If verified, updates the user's `subscriptions` record, resolves linked `referrals`, and triggers Resend onboarding/upgrade emails.
3. **`health-check`**: Validates Supabase PostgreSQL responses, Gemini API calls, and Groq Whisper endpoints, returning latency measurements.
4. **`ai-interview-chat`**: Feeds the current conversation history to the active LLM router. Injects historical context fetched from `match_transcript_embeddings` (RAG). Saves Q&As.
5. **`analyze-answer`**: Processes candidates' answers, tracking STAR logic, counting filler words (e.g. *like*, *ah*, *um*, *basically*), and outputting specific coaching recommendations.
6. **`analyze-resume`**: Downloads the PDF resume from the Supabase bucket via temporary signed URL, extracts text, and evaluates keywords.
7. **`ats-analyzer`**: Runs parsing prompts on resume text, returning matching profiles, missing keywords, and custom actionable roadmaps.
8. **`cleanup-stale-sessions`**: Set as a scheduled task. Selects active `interview_sessions` created $> 2$ hours ago and updates their status to `completed`.
9. **`embed-session`**: Generates a text embedding of candidate answers via Gemini `text-embedding-004`. Updates the candidate's `profiles` skill matrix.
10. **`fetch-resources`**: Resolves learning gaps to curate study links.
11. **`generate-feedback`**: Analyzes complete sessions, computing overall scorecard benchmarks, and records final reports in `feedback_reports`.
12. **`generate-prep-plan`**: Compiles skills memory parameters to generate a 7-day study plan.
13. **`send-email`**: Integrates Resend to send transaction records and notification reports.
14. **`send-streak-reminders`**: Checks practice streaks on profiles and dispatches notifications via email.
15. **`start-interview`**: Updates session states to active, checking monthly allocations.
16. **`transcribe-audio`**: Decodes incoming base64 audio streams and transcribes them via Groq's Whisper API.
17. **`vapi-webhook`**: Captures voice webhooks from external Vapi voice services, transcribing responses, and triggers evaluations.
18. **`mock-interview`**: Legacy text-only mock interview service endpoint.

## Section 3 — Frontend Client Design (Pages, Components & Hooks)

### 1. Routes Tree (`src/App.tsx`)
* `/` -> `Index.tsx` (Marketing Landing Page)
* `/auth` -> `Auth.tsx` (Sign-in / sign-up authentication)
* `/dashboard` -> `Dashboard.tsx` (User landing dashboard, statistics, and prep plans)
* `/interview/:sessionId` -> `InterviewSession.tsx` (Live interview room, voice flow, VAD indicators)
* `/report/:sessionId` -> `Report.tsx` (Interactive feedback report and visual scores)
* `/ats` -> `AtsAnalyzer.tsx` (Resume uploader and missing keywords checklist)
* `/pricing` -> `Pricing.tsx` (Plan grids and Razorpay checkout triggers)

### 2. Live Interview Room State Controller (`InterviewSession.tsx`)
* **Camera Feed**: Accesses browser device streams using `navigator.mediaDevices.getUserMedia`. If denied, switches UI layout to render a fallback profile badge card.
* **Auto-Submit Loop**: Listens to the `useTurnDetection` hook. When the turn state transitions to `complete`, it triggers `handleSendVoiceMessage()`, appending the user's transcript and invoking the AI chat endpoint.
* **Offline Protection**: Utilizes `useOnlineStatus` to monitor network states. If the client goes offline, speech activity parameters are held locally, displaying a warning banner.

### 3. Custom Voice Hooks

#### `useVoiceInput.ts`
Manages audio streams, speech APIs, and Whisper fallback conversions:
* **Real-time VAD**: Evaluates the root-mean-square (RMS) energy of browser audio inputs:
  $$\text{RMS} = \sqrt{\frac{1}{N} \sum_{i=1}^{N} x_i^2}$$
  Sets `audioEnergy` as a scaled normalized float ($0.0 \text{ to } 1.0$) for frontend visualizations.
* **Fallback Recording**: On iOS/Firefox, initializing the Web Speech API fails. The hook falls back to a `MediaRecorder` instance. Audio chunks are accumulated as standard WebM/MP4 blobs. When the VAD detects silence, the recorder is stopped, sending the base64-encoded audio chunk to the backend `transcribe-audio` function.

#### `useVoiceSynthesis.ts`
Speaks AI questions using `window.speechSynthesis`:
* **Chrome 15s Bug Fix**: Chrome SpeechSynthesis cuts out silently after speaking continuously for 15 seconds. To prevent this, the hook splits text strings into smaller chunks ($< 180$ characters) on sentence boundaries (`. `, `! `, `? `). It queues each chunk sequentially.
* **iOS Unlocking**: iOS silences all web-audio synthesis unless triggered directly by user gesture. The hook plays a silent WAV loop on a hidden audio element during the user's initial microphone tap.

#### `useTurnDetection.ts`
Maintains the state machine of the candidate's turn lifecycle:
* **State Machine**: Transitions through states: `idle` -> `listening` -> `active_speech` -> `waiting` -> `confirming` -> `complete`.
* **Acoustic Detection**: Scans the text stream for ending phrases.
* **Confirming Ratio**: When the active silence duration exceeds 80% of the computed threshold, the state transitions to `confirming` and renders the *"Keep Listening"* escape hatch.

## Section 5 — Mathematical & Algorithmic Designs

### 1. RAG Vector Search Metric
For RAG, the database calculates cosine similarity rather than Euclidean distance to compare vectors:
$$\text{Similarity}(A, B) = \cos(\theta) = \frac{A \cdot B}{\|A\| \|B\|}$$
Implemented in PostgreSQL via `1 - (embedding <=> query_embedding)` using pgvector operators.

### 2. User Skill Memory Update Formula
To track candidate progress over time, the system uses an Exponential Moving Average (EMA) to prevent individual poor sessions from zeroing out the candidate's historical progress:
$$S_{new} = \alpha \times S_{session} + (1 - \alpha) \times S_{old}$$
Where:
* $S_{new}$ = Newly stored skill rating.
* $S_{session}$ = Candidate score achieved in the current interview session.
* $S_{old}$ = Historical skill rating.
* $\alpha$ = Sensitivity decay multiplier. Confera uses $\alpha = 0.7$, weighting the current session heavily while preserving 30% of historical context.

## Section 6 — Auth & Token Lifecycle

1. **Token Generation**: Upon passwordless signup (OTP), Supabase Auth generates a JWT containing user claims and expiration parameters.
2. **Local Persistence**: The client stores the JWT session inside local browser storage.
3. **Request Inclusion**: Axios or Fetch interceptors append the token to all API calls:
   `Authorization: Bearer <jwt_token>`
4. **Edge Verification**: Supabase Edge Functions decrypt the token signature. The middleware retrieves the claims and matches the unique ID (`auth.uid()`) to restrict database query responses.

## Section 9 — Error Handling, Circuit Breakers & Retries

* **LLM Calls**: If an Edge Function receives a timeout or rate limit error, the shared service retries up to 3 times using exponential backoff:
  $$\text{Delay} = \text{RETRY\_BACKOFF\_MS} \times 2^{\text{attempt}}$$
* **Circuit Breakers**: If the retries fail, the breaker state changes to `open` for 2 minutes. Future calls route to the backup LLM provider immediately.
* **Frontend Error Boundaries**: The dashboard and interview routes are wrapped in an `ErrorBoundary` React component. If an audio node crashes, the client renders a fallback UI panel allowing the user to download logs and refresh.

## Section 10 — Performance Optimization & Memory Management

* **PDF Parsing**: Performed on the client side using `pdfjs-dist` worker tasks. This prevents large binary file uploads to the edge functions, saving bandwidth and processing time.
* **Lazy Loading**: Route pages are loaded dynamically using `React.lazy` and `Suspense`, reducing the initial JS bundle size.
* **Mic Stream Disposal**: The `cleanup()` routine inside `useVoiceInput` explicitly calls `.stop()` on all tracked media stream tracks to prevent memory leaks and keep-alive mic indicators in the browser.
* **Deno Instance Warmup**: The frontend triggers a silent GET call to the `health-check` endpoint during dashboard mounting to warm up cold edge function isolates before the candidate starts their interview.
